Object-Oriented Programming Basics

The notion of object-oriented programming is based upon the idea of computer programs arising from the transmission of messages sent between well-designed objects containing both behavior (methods) and state (data structures). Things like calculations, data manipulation and input/output operations are all handled by objects that have been specifically designed to perform this or that task or to provide a certain piece of information to another object. In most object-oriented languages, such as C++, Java, Ruby and Python, objects are class-based, meaning that individual objects are instances of particular classes, which are in turn themselves objects. Although built-in method behaviors can be overwritten, the behavior of objects which are instances of classes is always determined to some degree by the method definitions provided in the parent class, which are typically available in a language's official documentation.

WHAT ROLE DO OBJECTS PLAY IN JAVASCRIPT?

Like objects in other langagues, JavaScript objects are useful for collecting and managing data, and thus for modeling objects found in the real world. A JavaScript object is in fact a container of properties consisting of keyed collections of names and values, which makes them reminiscent of key-value pairs in Ruby hashes. JavaScript objects can be easily created with simple literal notation using open curly brackets. They can store and return data, and they can be nested inside other objects.

JavaScript Objects
  • Class-free
  • Keyed collections of names and values
  • Property inheritance by prototype linkage
